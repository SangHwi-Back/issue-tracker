## Team Workflow

- 첫 화면은 페어 프로그래밍. 클래스 선언만 하고 내부 로직은 개인 역량으로.
   - 각자 작업한 코드를 이해하지 못하는 상황을 최대한 줄이기 위함.
- feature branch를 merge 하고 싶을 땐 PR을 통해서 팀원이 확인 후 merge.
- 평일 10:00 ~ 데일리 스크럼 & 진행 공유 및 이슈 생성.
- 리뷰어 피드백 반영은 공유 후, hotfix branch 만들어서 진행.
